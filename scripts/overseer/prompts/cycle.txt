Continue as the Loose Cannon development overseer in this repository.

## Start every cycle
1. Read AGENTS.md, docs/STATUS.md, docs/MASTER_PLAN.md, and the latest docs/OVERSEER_LOG.md entries.
2. Inspect git status / recent changes if useful so you do not redo work.
3. Choose the single next incomplete milestone item from MASTER_PLAN (highest priority unfinished).
   Current expected order after 2026-07-10 refresh:
   - M3 memorial wall (if incomplete)
   - M3 goon stats feel
   - M5 combat/AI or pathing around shells
   - M6 more missions / presentation polish
   - M4 parties only when solo loop is solid
4. If STATUS lists a critical player-facing bug (movement, crashes, unreadable UI), fix that first.

## Product context (do not regress)
- 18+ crime game: strong language, violence, suggestive club content OK.
- Mode A: in-memory server only — NO Postgres, Redis, k8s, real auth.
- Free outdoor roam is intentional (district rep is advisory; do not re-add soft-kicks that block walking).
- The Titty Twister club, combat-scene graphics, longer kill toasts, mobile full-screen dialogue are live — preserve them.

## Execute
- Implement that milestone (or a coherent vertical slice if the full item is too large for one cycle).
- Spawn subagents when useful (explore before large edits; plan for multi-file systems).
- Keep server authority: clients send intents only.
- Protocol changes: packages/shared first, then server, then client.

## Verify
- Prefer: `npm run smoke` (server must already be on :3001, or start it)
- After structural/protocol changes: `npm run build`
- Fix breakages you introduced before ending the cycle.

## Close the cycle
1. Update docs/STATUS.md (accurate live status + date).
2. Append a cycle entry to docs/OVERSEER_LOG.md (focus, done, verify, next, blocked).
3. If MASTER_PLAN checkboxes should flip, update them.
4. End with a short summary: what shipped, what's next, any human decision needed.

If blocked on a product/design decision or external credentials, write the blocker clearly in STATUS and OVERSEER_LOG and stop implementing unrelated features.
